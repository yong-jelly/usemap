# 맛탐정 - 맛집 큐레이션 플랫폼

> 흩어진 맛집 정보를 한곳에 모아 체계적으로 관리하고, 신뢰할 수 있는 소스에서 맛집을 발견하는 서비스

## 왜 이 프로젝트를 시작했나?

### 문제 1: 저장은 쉽지만, 관리는 불가능

네이버 지도의 별표시, 폴더 저장, SNS 좋아요와 북마크... 수십, 수백 번 반복하다 보면 어느새 천 개가 넘는 맛집 목록이 쌓인다. 하지만 정작 필요할 때 꺼내 쓸 수가 없다.

- **출처를 잃어버린다**: 누가, 언제, 어디서 추천했는지 기억나지 않는다
- **저장한 이유를 잊는다**: 왜 이 음식점을 저장했는지 맥락이 사라진다
- **재탐색이 불가능하다**: 여행을 갈 때 특정 지역의 저장 목록을 한번에 볼 수 없다
- **결국 다시 검색한다**: 저장해둔 목록이 아니라 그냥 가까운 곳을 새로 검색하게 된다

### 문제 2: 네이버 지도 리뷰와 별점의 신뢰도 문제

- 광고 음식점이 상위에 노출된다
- 대부분의 리뷰가 이벤트/마케팅 업체를 통해 작성되어 신뢰도가 낮다
- 진짜 맛집을 찾기 위해 커뮤니티, 유튜브, SNS를 따로 뒤져야 한다

### 문제 3: 협업과 공유의 한계

- 네이버 폴더는 공유 기능이 제한적이다
- 회사 회식 장소 리스트, 친구들과의 공동 맛집 목록 관리가 어렵다
- 폴더 구독 개념이 없어 업데이트를 실시간으로 확인하기 어렵다
- 네이버 계정에 종속되어 장기 사용에 불안정하다

### 문제 4: 유튜브 맛집 정보의 접근성 문제

유튜브는 텍스트 리뷰나 사진보다 **훨씬 생생하게 음식점의 분위기와 음식을 전달**한다. 특히 맛집 전문 인플루언서들의 콘텐츠는 신뢰도 높은 추천 소스다.

**하지만 현실적인 문제가 있다:**

- 인기 먹방 채널은 구독자 수십만, 영상은 수백~수천 개에 달한다
- 나와 취향이 맞는 인플루언서를 찾아도 **과거 영상의 음식점을 하나씩 찾아보는 건 현실적으로 불가능**하다
- 영상을 보다가 "여기 가고 싶다" 생각해도 어딘가에 따로 메모해두기 번거롭다
- 결국 유튜브에서 본 맛집 정보는 **휘발**되어 사라진다

> "성시경이 추천한 음식점 어디였지?", "먹방 유튜버가 갔던 그 국밥집이 어디더라?"

**→ 이 플랫폼은 유튜브 채널별 맛집 정보를 체계적으로 목록화한다. 누군가 한 번 구축해놓으면 모두가 활용할 수 있는 가치 있는 데이터베이스가 된다.**

---

## 어떻게 해결하는가?

### 1. 신뢰할 수 있는 추천 소스 통합

광고와 마케팅이 범람하는 네이버 리뷰 대신, 실제 사용자들이 진심으로 추천하는 소스들을 통합한다.

| 소스 타입 | 설명 | 예시 |
|-----------|------|------|
| **유튜브** | 맛집 인플루언서의 모든 영상 속 음식점을 DB화 | 성시경, 먹방 채널 등 |
| **인스타그램** | 지역 맛집 인플루언서의 모든 게시물 속 음식점을 DB화 | 지역 맛집 계정 등 |
| **커뮤니티** | 오래된 커뮤니티의 검증된 맛집 추천 | 클리앙, 다모앙, 보배드림 |
| **네이버 플레이스 폴더** | 플레이스 에디터들의 테마별 큐레이션 | 테마별 맛집 리스트 |
| **맛탐정 폴더** | 일반 사용자가 직접 만드는 공개 큐레이션 | 동네 맛집, 회식 장소 등 |

> **핵심**: 유튜브, 인스타그램 등 외부 소스는 **플랫폼 관리자가 지속적으로 추적하며 데이터를 축적**한다. 한 번 구축된 데이터베이스는 모든 사용자에게 가치를 제공하는 **공공재**가 된다.

#### 유튜브/인스타그램 데이터베이스의 가치

수십만 팔로워를 가진 맛집 인플루언서의 콘텐츠를 생각해보자:

- **유튜브**: 영상으로 음식과 분위기를 생생하게 전달. 텍스트 리뷰보다 훨씬 많은 정보를 담고 있다.
- **인스타그램**: 특정 지역을 대표하는 맛집 계정이 있다. "부산 맛집", "제주 로컬 맛집" 등.

**문제는 동일하다:**
- 과거 콘텐츠 속 음식점을 하나씩 찾아보는 건 **현실적으로 불가능**
- 취향이 맞는 인플루언서를 찾아도 그 사람의 모든 추천을 저장할 방법이 없다

**이 플랫폼의 해결책:**
- 관리자가 인플루언서의 콘텐츠를 지속적으로 추적하며 음식점 정보를 DB화
- 사용자는 채널/계정을 구독하면 **모든 음식점 목록을 한눈에** 확인
- 지역별 필터링으로 "이 유튜버가 간 강남 맛집"만 추출
- 원본 콘텐츠 링크와 함께 저장되어 언제든 다시 확인 가능

### 2. 맛탐정: 누구나 큐레이터가 될 수 있다

**"맛탐정"은 특별한 사람이 아니다. 바로 당신이 될 수 있다.**

일반 사용자가 공개 폴더를 만들고 자신만의 맛집 큐레이션을 시작하면, 그 순간부터 맛탐정이 된다.

```
🕵️ 맛탐정 @미식가김철수
├── 📂 결혼기념일 추천 레스토랑 (구독자 234명)
├── 📂 부모님과 함께하기 좋은 곳 (구독자 156명)
├── 📂 아이들과 가기 좋은 식당 (구독자 89명)
└── 📂 혼밥하기 좋은 곳 (구독자 412명)
```

#### 맛탐정이 되면 얻는 것

1. **구독자 확보**: 취향이 맞는 다른 사용자들이 폴더를 구독한다
2. **신뢰도 축적**: 꾸준히 좋은 추천을 하면 구독자가 늘어나고, 플랫폼 내 영향력이 생긴다
3. **전문성 인정**: "이 사람이 추천하면 믿을 수 있다"는 평판이 쌓인다
4. **커뮤니티 기여**: 자신의 미식 경험이 다른 사람들에게 실질적인 도움이 된다

> 네이버 지도에 그냥 별표시만 하던 사람이, 이제는 **자신의 맛집 철학을 담은 큐레이션**을 만들고 공유할 수 있다.

### 3. 폴더 기반 체계적 관리

```
📁 나의 폴더
├── 📂 서울 데이트 코스 (public - 맛탐정 폴더)
│   ├── 🍽️ 성수동 파스타집 - "분위기 좋고 음식도 맛있어요"
│   ├── 🍽️ 한남동 오마카세 - "예약 필수, 가격대비 최고"
│   └── 🍽️ 이태원 타코집 - "2차로 가기 좋음"
├── 📂 회사 점심 (invite - 동료들과 공동 편집)
│   └── 🍽️ ...
└── 📂 제주도 여행 (private - 나만 보기)
    └── 🍽️ ...
```

- **메모 기능**: 저장한 이유, 추천 포인트, 개인적인 노트 추가
- **출처 추적**: 어디서 발견했는지 자동 기록 (커뮤니티/유튜브/SNS)
- **권한 관리**: public(공개), private(비공개), invite(초대 코드) 모드
- **협업 편집**: 폴더를 함께 편집할 수 있는 참여형 폴더

### 4. 소스 등록과 교차 검증: 신뢰도의 핵심

**이 플랫폼의 핵심은 "소스(출처)"다.**

사용자는 음식점 상세 페이지에서 **관련 콘텐츠**를 직접 등록할 수 있다:
- 유튜브 영상 링크
- 인스타그램/쓰레드 게시물
- 커뮤니티 글 링크
- 블로그 리뷰 등

#### 교차 검증 메커니즘

```
🍽️ 강남 스시오마카세
├── 📺 유튜브: 성시경 채널 (2024.03)
├── 📺 유튜브: 먹방 유튜버A (2024.01)
├── 💬 커뮤니티: 클리앙 맛집게시판 (2023.12)
├── 📸 인스타그램: @seoul_foodie (2024.02)
└── 🕵️ 맛탐정: @미식가김철수 폴더에 포함
```

**하나의 음식점에 여러 소스가 쌓이면:**

1. **객관성 확보**: "이 유튜버만 추천한 게 아니라, 커뮤니티에서도 인정받은 곳이구나"
2. **깊은 이해**: 유튜브 링크가 있으면 사진/텍스트만으로는 알 수 없는 **분위기, 서비스, 실제 음식**을 영상으로 확인
3. **신뢰도 시각화**: 소스가 많을수록 "검증된 맛집"으로 인식

> 맛탐정이 추천한 음식점 + 여러 소스의 교차 검증 = **광고 리뷰와는 차원이 다른 신뢰도**

### 5. 구독 기반 피드 시스템

관심 있는 소스(유튜브 채널, 인스타 계정, 커뮤니티 지역, 맛탐정 폴더)를 구독하면 새로운 맛집이 추가될 때 피드로 받아볼 수 있다.

- 최신순 / 거리순 정렬
- 그리드 / 리스트 뷰 전환
- 위치 기반 필터링

### 6. 지역별 탐색

여행을 갈 때, 특정 지역의 모든 저장된 맛집을 한번에 확인할 수 있다.

- 지역별 맛집 리스트업
- 지도 뷰로 한눈에 확인
- 소스별 필터링 (커뮤니티만, 유튜브만 등)

## 핵심 기능

### 구현 완료

## 타겟 사용자

1. **맛집 정보를 열심히 모으지만 관리가 안 되는 사람**
2. **광고성 리뷰가 아닌 진짜 추천을 찾는 사람**
3. **좋아하는 유튜버의 맛집 정보를 한 곳에서 보고 싶은 사람**
4. **친구/동료와 맛집 리스트를 공유하고 싶은 사람**
5. **여행 시 특정 지역 맛집을 효율적으로 탐색하고 싶은 사람**

## 핵심 가치 제안

> **"흩어진 맛집 정보의 허브, 그리고 교차 검증 플랫폼"**

### 플랫폼이 제공하는 가치

1. **소스 데이터베이스** (관리자 구축)
   - 유튜브/인스타그램 인플루언서의 모든 맛집 정보를 DB화
   - 커뮤니티 맛집 추천글 자동 수집
   - 한 번 구축되면 모든 사용자가 활용하는 공공재

2. **맛탐정 생태계** (사용자 참여)
   - 누구나 자신만의 맛집 큐레이션을 공개하고 구독자를 모을 수 있음
   - 구독자 수와 피드백을 통해 신뢰도와 전문성이 축적됨

3. **교차 검증 시스템** (신뢰도 확보)
   - 하나의 음식점에 여러 소스가 연결됨
   - 맛탐정 추천 + 유튜브 + 커뮤니티 = 광고 리뷰와는 차원이 다른 객관성
   - 유튜브 링크를 통해 텍스트로는 알 수 없는 생생한 정보 제공

### 네이버 지도와의 근본적 차이

| | 네이버 지도 |맛탐정 |
|---|-------------|--------|
| **추천 소스** | 광고 + 이벤트 리뷰 | 인플루언서, 커뮤니티, 맛탐정 |
| **신뢰도** | 마케팅 업체가 만든 리뷰 | 교차 검증된 다중 소스 |
| **큐레이션** | 없음 | 맛탐정의 테마별 폴더 |
| **영상 정보** | 없음 | 유튜브 링크로 생생한 정보 |
| **사용자 역할** | 소비자 | 소비자 + 큐레이터(맛탐정) |
