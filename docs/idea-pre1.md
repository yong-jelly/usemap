# 맛탐정 UI/UX 개선을 위한 핵심 기획 및 가이드라인

이 문서는 기존의 단순 이미지 나열 방식에서 벗어나, 서비스의 핵심 가치인 **'맥락 중심의 저장과 관리'**를 실현하기 위한 UI/UX 개선 방향을 정리한 문서입니다.

---

## 1. 핵심 철학: "맥락(Context)의 보존과 복구"

### 1.1 문제 정의
- **맥락 소실(Context Loss)**: 네이버 지도, SNS 북마크 등 기존 저장 방식은 '장소'는 남지만 '왜, 어디서' 저장했는지에 대한 기억이 사라짐.
- **막연한 나열**: 홈 화면이 단순 이미지 위주로 구성되어 있어, 사용자가 자신의 저장 의도를 즉각적으로 떠올리기 어려움.

### 1.2 해결 방향
- **미식 아카이브(Archive) 메타포**: 단순 SNS가 아닌, 나만의 미식 지식 베이스(Knowledge Base)로 포지셔닝.
- **Slack '나중에' 감성**: "지금은 저장하고, 나중에 그 근처에 갔을 때 저장 당시의 맥락을 완벽히 복구"하는 도구적 가치 제공.

---

## 2. UI/UX 개선 상세 전략

### 2.1 홈 화면 (HomePage) 메타포 전환
- **SNS 피드형 → 대시보드/아카이브형**
- **위치 기반 시작점 제공**: 사용자의 현재 위치나 관심 지역을 기반으로 '지금 당장 유효한 맥락'을 우선 노출.
- **상태 브리핑**: "강남역 주변에 당신이 저장한 맥락이 3개 있습니다"와 같은 행동 유도형 문구 활용.

### 2.2 장소 카드 (Place Card) 디자인 개편
- **이미지 중심 → 정보 밀도 중심**
- **맥락 바(Context Bar) 도입**: 카드 상단/하단에 출처(유튜브 채널명, 인스타 계정 등)를 명확히 표시.
- **메모 전면화**: 사용자가 작성한 메모를 이미지만큼 중요한 비중으로 노출하여 '저장 의도'를 즉각 인지시킴.
- **검증 마크**: 여러 소스(유튜브, 커뮤니티 등)에서 교차 검증된 경우 '신뢰 지표'를 시각적으로 표현.

### 2.3 탐색 가이드 (Discovery Guide)
- **이동 경로 기반 추천**: 사용자가 이동하며 탐색한 과거 경로를 기억하여, 해당 지역 재방문 시 관련 맥락을 리마인드.
- **발견의 테마화**: 막연한 나열 대신 #혼밥 #성시경맛집 #비오는날 등 테마별 큐레이션 제공.

## 3. 개선 TODO 리스트

### [Phase 1] 맥락 가시성 강화 (Quick Wins)
- [ ] **장소 카드 레이아웃 수정**: 메모와 출처(Source)가 이미지와 함께 1:1 비중으로 보이도록 조정.
- [ ] **출처 아이콘/배지 추가**: 유튜브, 인스타, 커뮤니티 등 소스 타입을 직관적으로 표시.
- [ ] **홈 상단 위치 가이드 배너**: 현재 위치 설정 유도 및 주변 맛집 맥락 브리핑 섹션 추가.

### [Phase 2] 도구적 기능 고도화 (Core Features)
- [ ] **위치 기반 필터링 강화**: `/feed?sort=distance` 기능을 홈 화면의 핵심 모드로 통합.
- [ ] **맥락 입력 유도 UX**: 장소 저장 직후 "왜 저장하시나요?"와 같은 짤막한 메모 입력을 유도하는 인터랙션 추가.
- [ ] **미니 맵(Mini-map) 연동**: 카드 내 혹은 섹션 옆에 해당 장소의 위치적 맥락을 보여주는 작은 지도 조각 배치.

### [Phase 3] 아카이브 가치 극대화 (Advanced)
- [ ] **이동 경로 기반 리마인드**: 과거 탐색 경로 데이터를 활용한 맞춤형 홈 피드 구성.
- [ ] **교차 검증 대시보드**: 하나의 장소에 쌓인 여러 소스(유튜브+커뮤니티+인스타)를 한눈에 비교 분석할 수 있는 상세 뷰.
- [ ] **폴더 메타포 시각화**: 폴더를 '책장'이나 '서류함'처럼 디자인하여 축적의 가치 시각화.

---

## 4. 디자인 원칙 (Design Principles)
1. **Actionable**: 모든 정보는 "나중에 방문할 때" 도움이 되어야 한다.
2. **Traceable**: 모든 정보는 "어디서 왔는지" 추적 가능해야 한다.
3. **Personal**: 남의 취향보다 "나의 메모와 나의 맥락"이 우선시되어야 한다.
4. **Reliable**: 다양한 소스의 교차 검증을 통해 "나쁘지 않다는 확신"을 주어야 한다.
