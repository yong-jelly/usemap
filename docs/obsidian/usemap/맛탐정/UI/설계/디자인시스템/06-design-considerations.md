# 6. 디자인 고민 및 결정

모바일 앱 UI/UX 재설계 시 검토해야 할 핵심 이슈와 결정 방향.

## 6.1 정보 계층 (Information Hierarchy)

### 출처 vs 맛 정보

- **결정**: 출처를 우선 노출. 맛/별점은 맛탐정에서 제공하지 않음.
- **고민**: 사용자가 "맛있나?"를 가장 먼저 묻는데, 맛탐정은 출처로 신뢰를 대체함.
- **해결**: 출처 개수, 콜렉션 저장 수, 저장 인원 수로 "많이 언급됐다"를 전달. 네이버 지도 버튼으로 외부 확인 유도.

### 장소 카드 정보 밀도

- **고민**: 카드에 넣을 정보 vs 상세에서만 노출.
- **방향**: 카드에는 제목, 지역, 카테고리, 출처 요약(배지), 액션(좋아요/저장). 상세에서 메모·댓글·출처 상세.

---

## 6.2 네비게이션 구조

### 탭 바 vs 드로어

- **결정**: 하단 탭 바 4개 (홈, 피드, 맛탐정, 프로필).
- **이유**: 모바일에서 빈번한 전환, 한 손 조작, Apple HIG 권장.

### 홈 vs 피드 vs 맛탐정 구분

- **홈**: 추천·소스 탐색·구독 탭. 첫 진입점.
- **피드**: 구독한 소스의 최신 장소 피드.
- **맛탐정**: 공개 폴더 탐색, 구독 가능 소스.
- **고민**: 홈과 맛탐정이 겹칠 수 있음. "추천"과 "탐색"을 명확히 구분할지 검토 필요.

---

## 6.3 저장 플로우 (좋아요 vs 콜렉션)

### 2단계 액션

- **1단계**: 좋아요 (가벼운 관심 표시)
- **2단계**: 콜렉션 저장 (폴더 선택, 메모)
- **고민**: 좋아요만 하고 콜렉션에 안 넣는 사용자 비율이 높을 수 있음.
- **방향**: 좋아요 목록을 "나중에 정리" 진입점으로 활용. 주기적 "콜렉션에 넣기" 유도.

### 폴더 선택 시트

- **고민**: 폴더가 많을 때 스크롤 부담, 검색 필요 여부.
- **방향**: 최근 사용 폴더 상단, "새 폴더 만들기" 항상 노출.

---

## 6.4 출처 시각화

### Source Badge 디자인

- **유튜브**: 빨간 계열 (브랜드 연상)
- **커뮤니티**: 파란 계열
- **맛탐정 폴더**: Primary
- **고민**: 출처가 여러 개일 때 "2개" 숫자만 vs 아이콘 나열.
- **방향**: 숫자 + 툴팁/상세에서 개별 링크. 카드 공간 절약.

---

## 6.5 모바일 성능 vs 시각적 품질

### backdrop-blur 금지

- **결정**: 모바일에서 `backdrop-blur` 사용 금지.
- **이유**: 스크롤 시 GPU 부하, 프레임 드랍.
- **대안**: 단색 오버레이, 그라디언트.

### 애니메이션

- **결정**: transition 최소화. 필수 인터랙션에만.
- **이유**: 리스트·카드가 많을 때 합성 레이어 비용.

---

## 6.6 접근성

### 터치 영역

- **최소 44×44px** (Apple HIG).
- **고민**: 카드 내 좋아요/저장 아이콘을 24px로 할 때 터치 영역 확장 필요.
- **방향**: 아이콘 주변 패딩으로 터치 영역 44px 확보.

### 색상 의존

- **결정**: 출처 구분에 색상만 의존하지 않음. 아이콘 + 텍스트 병행.
- **이유**: 색맹 사용자.

---

## 6.7 다크 모드

- **고민**: 맛탐정이 음식·이미지 중심일 때 다크 모드 필요성.
- **방향**: 1차 출시는 라이트 모드만. 다크 모드는 토큰 구조로 준비 후 추후 지원.
