# 13. Presentation Design

맛탐정 투자 유치·파트너십을 위한 키노트 프레젠테이션 설계.

---

## Meta

| 항목 | 내용 |
|------|------|
| Topic | 맛탐정 — 출처 기반 맛집 발견 플랫폼 |
| Audience | 투자자 / 초기 파트너 |
| Duration | 20분 (15분 발표 + 5분 Q&A) |
| Objective | PERSUADE — 투자 및 파트너십 설득 |
| Slide Count | 25장 |
| Tool | Keynote / Google Slides / Figma |

---

## 1. NARRATIVE ARCHITECTURE

### 1.1 Story Arc (Hero's Journey)

```text
ACT 1: 세계의 문제 (Slides 1~6)
"모든 사람이 맛집을 저장하지만, 아무도 그 목록을 쓰지 못한다."
→ 공감 유발 + 문제의 크기 제시

ACT 2: 해결과 증명 (Slides 7~16)
"출처를 바꾸면, 발견이 달라진다."
→ 솔루션 시연 + 차별점 + 시장 기회 + 트랙션

ACT 3: 함께할 미래 (Slides 17~25)
"이 시장의 첫 번째 플레이어가 됩니다."
→ 비즈니스 모델 + 로드맵 + 팀 + Ask
```

### 1.2 Opening Hook (첫 60초)

> "지난 주말, 여러분은 맛집을 검색하셨을 겁니다. 네이버 지도를 열고, 별점 4.5 이상을 필터링하고, 리뷰를 읽었죠. 그리고 도착해보니 — 광고 음식점이었습니다."
>
> "이것은 한국 3,200만 네이버 지도 사용자 중 대다수가 경험하는 일입니다."
>
> "저장한 맛집은 수백 개인데, 정작 근처에 갔을 때 '여기 왜 저장했지?'라고 생각해본 적 있으시죠?"
>
> "오늘 맛탐정이 이 문제를 어떻게 바꾸는지 보여드리겠습니다."

### 1.3 Key Message Hierarchy (3개)

| 순서 | 핵심 메시지 | 전달 슬라이드 |
|------|------------|-------------|
| 1 | **출처가 다르면, 발견이 다르다** — 별점이 아닌 신뢰할 수 있는 출처(유튜브·커뮤니티·큐레이터)로 맛집을 발견하는 새로운 패러다임 | 7~9 |
| 2 | **맥락이 남으면, 저장이 활용된다** — 누가, 어디서, 왜 추천했는지 맥락이 보존되어 "저장한 목록이 실제로 쓰이는" 경험 | 10~11 |
| 3 | **누구나 맛탐정이 될 수 있다** — 소비자 → 큐레이터 전환으로 UGC 기반 소스 생태계 성장 | 12~13 |

### 1.4 Closing Call-to-Action

> "맛탐정은 '맛집 검색'을 '맛집 발견'으로 바꿉니다. 별점 경제를 출처 경제로 전환하는 첫 번째 플랫폼에 함께해 주십시오."

### Slide 04 — The Problem (2/3): 저장의 무덤

| 항목 | 내용 |
|------|------|
| Layout | Data (아이콘 + 수치 3열) |
| Visual | 다크 배경. 3개의 데이터 포인트가 가로 정렬. 각각 아이콘 + 숫자 + 라벨. |
| Headline | **저장은 쉽다. 활용은 불가능하다.** |
| Body (3열) | 📌 평균 **237개** 저장된 맛집 · 🔍 **83%** 저장 후 재방문 안 함 · ❓ **91%** "왜 저장했는지 기억 안 남" |
| Speaker Notes | "네이버 지도 사용자의 평균 저장 장소는 237개입니다. 하지만 83%는 저장 후 재방문하지 않습니다. 왜? 91%가 '왜 저장했는지 기억나지 않는다'고 합니다. 저장은 했지만, 맥락이 사라졌기 때문입니다. 누가 추천했는지, 어떤 영상에서 봤는지, 어떤 이유로 저장했는지 — 전부 사라집니다." |
| Animation | 3열 순차 slide-up (0.3s 간격) |

### Slide 05 — The Problem (3/3): 유튜브 정보의 휘발

| 항목 | 내용 |
|------|------|
| Layout | Quote + visual |
| Visual | 다크 배경. 왼쪽에 큰 따옴표. 오른쪽에 흐릿한 유튜브 플레이어 목업. |
| Headline | (없음) |

| Speaker Notes | "유튜브는 음식의 분위기와 맛을 가장 생생하게 전달합니다. 하지만 인기 먹방 채널의 영상은 수백~수천 개. '그 영상에서 본 그 집'을 다시 찾는 것은 현실적으로 불가능합니다. 유튜브 맛집 정보는 휘발됩니다. 보는 순간만 존재하고, 저장할 방법이 없습니다." |
| Animation | 따옴표 + 본문: fade-in → Sub-body: fade-in (0.5s delay) |

### Slide 06 — Opportunity: 시장

| 항목 | 내용 |
|------|------|
| Layout | Data (면적 차트 + 수치) |
| Visual | 다크 배경. 겹치는 원(Venn) 3개로 시장 규모 시각화. 각 원에 라벨. |
| Headline | **TAM → SAM → SOM** |
| Body | TAM: 네이버 지도 MAU **3,200만** · SAM: 맛집 적극 탐색자 **480만** · SOM: 초기 타겟 (20~40대, 출처 민감) **120만** |
| Speaker Notes | "한국 네이버 지도 월간 사용자 3,200만 명. 이 중 맛집을 적극적으로 탐색하는 사용자가 약 480만 명입니다. 맛탐정의 초기 타겟은 20~40대 중 광고 리뷰에 회의적이고, 유튜브·인스타그램으로 맛집 정보를 소비하는 120만 명입니다. 이들은 이미 '출처가 중요하다'는 것을 알고 있지만, 도구가 없었습니다." |
| Animation | TAM 원 fade-in → SAM 원 fade-in (0.3s) → SOM 원 scale-in (0.3s) |

| Animation | 목업: slide-up 0.5s → 텍스트: fade-in (0.3s delay) |

### Slide 09 — How It Works (3단계)

| 항목 | 내용 |
|------|------|
| Layout | Content (가로 3열, 아이콘 + 텍스트) |
| Visual | 밝은 배경. 3개 단계가 화살표로 연결. 각 단계에 아이콘 원. |
| Headline | **발견 → 검증 → 저장** |
| Col 1 | 🔍 **발견** · 출처 기반 피드에서 맛집을 만나다 |
| Col 2 | ✅ **검증** · 유튜브 + 커뮤니티 + 맛탐정 교차 확인 |
| Col 3 | 📂 **저장** · 맥락과 함께 폴더에 정리 |
| Speaker Notes | "사용 흐름은 3단계입니다. 첫째, 출처 기반 피드에서 맛집을 발견합니다. 구독한 유튜브 채널, 맛탐정 큐레이터, 커뮤니티에서 새로운 맛집이 올라옵니다. 둘째, 하나의 음식점에 연결된 여러 소스를 보고 교차 검증합니다. '유튜브 2개, 커뮤니티 3개, 맛탐정 5명이 추천' — 이 정보만으로도 판단이 됩니다. 셋째, 맥락과 함께 저장합니다. '어떤 유튜브에서 봤는지'가 자동 기록됩니다." |
| Animation | 3열 순차 slide-up (0.4s 간격) + 화살표 draw |

### Slide 10 — Key Benefit 1: 맥락 보존

| 항목 | 내용 |
|------|------|
| Layout | Split (좌: 비교, 우: 앱 목업) |
| Visual | 좌측에 Before/After 비교 박스. 우측에 장소 상세 화면 목업(출처 배지, 폴더명, 메모 표시). |
| Headline | **맥락이 남으면, 목록이 살아난다.** |
| Before | ❌ "여기 왜 저장했지?" |
| After | ✅ "유튜브 채널 X에서 본, 회식 폴더에 있는 국밥집" |
| Speaker Notes | "기존에는 장소 이름만 남고 맥락은 사라졌습니다. 맛탐정은 '어떤 유튜브에서 봤는지', '어떤 폴더에 저장했는지', '누가 추천했는지'가 자동으로 기록됩니다. 몇 달 후 그 근처에 갔을 때, 저장 당시의 맥락이 그대로 복구됩니다. 저장한 목록이 실제로 쓰이는 경험. 이것이 맛탐정의 핵심 가치입니다." |
| Animation | Before: fade-in → (0.5s) After: slide-up + 목업 동시 |

### Slide 11 — Key Benefit 2: 교차 검증

| 항목 | 내용 |
|------|------|
| Layout | Content (시각적 도해) |
| Visual | 중앙에 하나의 음식점. 주변에서 4개 소스(유튜브, 인스타, 커뮤니티, 맛탐정)가 연결선으로 들어오는 다이어그램. 소스가 많을수록 연결선이 두꺼워지는 표현. |
| Headline | **1개 소스: "글쎄." 4개 소스: "가보자."** |
| Body | 하나의 음식점에 여러 독립적 출처가 연결될수록 신뢰도가 올라가는 교차 검증 시스템. |
| Speaker Notes | "한 명이 추천하면 '글쎄요'입니다. 하지만 유튜브 채널 2개, 커뮤니티 게시글 3개, 맛탐정 큐레이터 5명이 모두 같은 음식점을 추천하면 — 가볼 만하다는 확신이 생깁니다. 이것은 광고 리뷰와 근본적으로 다릅니다. 서로 관련 없는 독립적 소스들의 합의이기 때문입니다." |
| Animation | 음식점 아이콘: instant → 4개 소스 순차 fly-in (0.3s 간격) → 연결선 draw |

| Animation | 목업: slide-up → 텍스트: fade-in |

## 5. PRESENTER GUIDELINES

### 5.1 Pacing

| 섹션 | 슬라이드 | 시간 | 비고 |
|------|----------|------|------|
| Opening & Problem | 01~05 | 4분 | 감정 유발. 여유 있게. |
| Opportunity | 06 | 1분 | 숫자만 빠르게. |
| Solution & Demo | 07~13 | 5분 | 핵심. 데모에 집중. |
| Competition & Traction | 14~16 | 3분 | 데이터 기반 논리적. |
| Future & Team | 17~21 | 3분 | 신뢰 구축. |
| The Ask & Closing | 22~25 | 2분 | 임팩트. 깔끔하게. |
| Q&A | — | 5분 | — |
| **합계** | **25장** | **~18분 + 5분 Q&A** | |

### 5.2 Transition Scripts

| 전환 | 스크립트 |
|------|---------|
| Problem → Opportunity (05→06) | "이 문제의 크기를 숫자로 보겠습니다." |
| Opportunity → Solution (06→07) | "시장은 충분합니다. 이제 맛탐정이 어떻게 해결하는지 보여드리겠습니다." |
| Solution → Demo (09→13) | "말로 설명하는 것보다 직접 보시는 게 빠릅니다." |
| Demo → Competition (13→14) | "이 솔루션이 시장에서 어디에 위치하는지 보겠습니다." |
| Traction → Future (16→17) | "지금까지 쌓아온 것입니다. 이제 어디로 가는지 말씀드리겠습니다." |
| Roadmap → Ask (21→22) | "이 계획을 실행하기 위해 필요한 것입니다." |
