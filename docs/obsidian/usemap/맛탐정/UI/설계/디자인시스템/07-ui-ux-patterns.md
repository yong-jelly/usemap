# 7. UI/UX Pattern Master

Apple HIG 기반, 맛탐정 모바일 앱 UI/UX 패턴 정의.

## User Research Insights

### Primary User (Persona)

> **민지, 29세, 서울 직장인**
> 유튜브·인스타에서 맛집을 발견하면 네이버 지도에 별표를 남기지만 수백 개가 쌓여 관리가 안 됨. 막상 근처 갔을 때 "왜 저장했지?" 기억이 나지 않아 결국 새로 검색함. 광고 리뷰에 지쳐 "진짜 추천"을 찾고 싶음.

### Top 3 User Goals

1. **발견**: 출처가 명확한 맛집을 빠르게 탐색하고 싶다
2. **저장**: 맥락(출처, 메모, 폴더)과 함께 체계적으로 관리하고 싶다
3. **활용**: 특정 지역에 갔을 때 저장한 맛집을 즉시 꺼내 쓰고 싶다

### Pain Points in Current Solutions

1. 네이버 지도: 별표만 가능, 출처·맥락 소실, 광고 리뷰
2. 유튜브/인스타: 정보는 풍부하나 저장·관리 불가, 검색 어려움
3. 맛집 앱(다이닝코드 등): 별점·리뷰 기반, 출처 불투명, 교차 검증 없음

## 1. HIERARCHY & LAYOUT

### 1.1 Visual Hierarchy Strategy

사용자가 화면을 볼 때 인지하는 순서를 의도적으로 설계함.

#### PlaceCard (장소 카드)

```text
+---------------------------------------+
| [1] 이미지 (16:9)                      |  ← 가장 먼저: 시각적 흥미
|                                       |
+---------------------------------------+
| [2] 장소명 (Title 2, Semibold)         |  ← 두 번째: "어디인가"
| [3] 지역 · 카테고리 (Subheadline)       |  ← 세 번째: "어디에 있고, 뭐 파는 곳인가"
| [4] 출처 배지 (Caption, 컬러)           |  ← 네 번째: "왜 믿을 수 있는가" (핵심 차별점)
| [5] ♡ 좋아요  ⊞ 저장 (아이콘)           |  ← 다섯 번째: 액션
+---------------------------------------+
```

| 순서 | 요소 | 인지 목적 | 크기/강조 |
|------|------|-----------|-----------|
| 1st | 이미지 | 시각적 hook, 음식 종류 판단 | 가장 큰 영역 |
| 2nd | 장소명 | 식별 | Title 2, Semibold |
| 3rd | 지역·카테고리 | 위치 판단, 음식 종류 확인 | Subheadline, muted |
| 4th | 출처 배지 | 신뢰 근거 (Traceable) | Caption, 컬러 코딩 |
| 5th | 액션 버튼 | 관심 표시 | 아이콘, 우하단 |

#### PlaceDetail (장소 상세)

```text
+---------------------------------------+
| [1] 이미지 갤러리                      |
+---------------------------------------+
| [2] 장소명 + 카테고리                   |
| [3] 지역 주소                          |
+---------------------------------------+
| [4] 출처 하이라이트                     |  ← 핵심 영역
|     유튜브 2  커뮤니티 1  폴더 3        |
|     콜렉션 저장 12명                    |
+---------------------------------------+
| [5] 액션 바                            |
|     ♡ 좋아요  ⊞ 저장  N 네이버  … 더보기 |
+---------------------------------------+
| [6] 탭: 메뉴 | 리뷰 | 출처             |
+---------------------------------------+
```

### 1.2 Scan Pattern

- **F-pattern**: 리스트 화면 (홈, 피드, 검색 결과). 좌측 정렬 제목 → 우측 메타/액션 순으로 스캔.
- **Z-pattern**: 카드 그리드 (소스 탐색, 폴더 목록). 이미지 → 제목 → 다음 카드 순.

### 1.3 Content Density

| 화면 | 밀도 | 이유 |
|------|------|------|
| 홈 추천 | Medium | 카드 간 여유, 이미지 강조, 탐색 모드 |
| 피드 | High | 빠른 스캔, 이미 구독한 소스의 새 장소 |
| 장소 상세 | Low~Medium | 정보 소화 시간 필요, 출처 영역 강조 |
| 검색 결과 | High | 빠른 비교, 컴팩트 카드 |
| 프로필 | Low | 개인 공간, 여유로운 레이아웃 |

> **Designer's Note**: 맛탐정의 핵심 가치는 "출처 기반 신뢰"이므로, 장소 상세에서 출처 하이라이트 영역은 의도적으로 여백을 충분히 두어 시각적 무게를 부여함. 일반적인 맛집 앱이 별점·리뷰 수를 강조하는 영역을 출처 정보로 대체한 것이 핵심 차별점.

## 2. PLATFORM-SPECIFIC PATTERNS

### 2.1 Navigation Pattern

**Tab Bar (하단 4탭) + Navigation Stack**

```text
+-----------------------------------+
|            Content Area           |
|                                   |
+-----------------------------------+
| [홈]   [피드]  [맛탐정]  [프로필]  |
+-----------------------------------+
```

| 탭 | 아이콘 | 역할 |
|----|--------|------|
| 홈 | Home | 추천·소스 탐색·구독 (3개 상단 탭) |
| 피드 | Rss/Bell | 구독 소스의 새 장소 피드 |
| 맛탐정 | Compass | 공개 폴더·소스 탐색 |
| 프로필 | User | 내 콜렉션·저장·방문·설정 |

**Navigation Stack**: 각 탭에서 카드 클릭 → 상세 모달 (push가 아닌 모달 방식).

> **Designer's Note**: 장소 상세를 모달로 처리하는 이유는 "스크롤 위치 보존" 때문. 리스트에서 카드를 눌러 상세를 보고 돌아왔을 때, 이전 스크롤 위치가 유지되어야 탐색 흐름이 끊기지 않음. 네이버 지도도 동일 패턴.

### 2.2 Modal Presentation Guidelines

| 모달 유형 | Presentation | 용도 |
|-----------|-------------|------|
| 장소 상세 | Full-screen sheet (drag to dismiss) | PlaceDetailModal |
| 폴더 선택 | Half sheet (60~80vh) | SaveToCollectionSheet |
| 확인 다이얼로그 | Center alert | 삭제 확인, 로그아웃 |
| 인증 | Center modal | AuthModal |
| 액션 시트 | Bottom sheet | PlaceActionSheet |
| 필터 | Bottom sheet | FilterBottomSheet |
| 위치 설정 | Bottom sheet | LocationSettingSheet |

### 2.3 Gesture Definitions

| 제스처 | 영역 | 동작 |
|--------|------|------|
| Swipe down | 장소 상세 모달 | Dismiss (닫기) |
| Pull-to-refresh | 홈, 피드, 소스 상세 | 데이터 갱신 |
| Swipe right | Navigation stack | 이전 화면 (iOS 기본) |
| Long press | PlaceCard | 컨텍스트 메뉴 (저장, 공유) |
| Tap | 카드, 버튼, 탭 | 기본 상호작용 |

### 2.4 Context Menus & Action Sheets

#### PlaceActionSheet

```text
+---------------------------------------+
| 리뷰 쓰기                              |
| 다녀왔어요                              |
| 관련 콘텐츠 추가                         |
| ─────────────────                     |
| 공유하기                                |
| ─────────────────                     |
| 취소                                   |
+---------------------------------------+
```

#### FolderListOptionsSheet

```text
+---------------------------------------+
| 리스트 수정하기                         |
| 공유하기                                |
| ─────────────────                     |
| 폴더 삭제 (관리자만)                     |
| ─────────────────                     |
| 취소                                   |
+---------------------------------------+
```

> **Designer's Note**: 액션 시트는 Apple HIG의 "Don't make me think" 원칙. 현재 맥락에서 가능한 행동만 노출하여 인지 부하를 줄임. 구분선으로 위험 액션(삭제)을 시각적으로 분리.

## 3. SCREEN DESIGNS

### Screen 1: Onboarding / Welcome

```text
+---------------------------------------+
|                                       |
|          [맛탐정 로고]                  |
|                                       |
|   흩어진 맛집 정보의 허브               |
|   출처 기반 발견 플랫폼                 |
|                                       |
|   ● ○ ○  (3 steps indicator)          |
|                                       |
|   +-----------------------------+     |
|   |        시작하기 (Primary)     |     |
|   +-----------------------------+     |
|                                       |
|   이미 계정이 있으신가요? 로그인         |
|                                       |
+---------------------------------------+
```

**Component Inventory**:
- 로고 (centered)
- 타이틀: Display, Bold
- 서브타이틀: Body, muted
- Page indicator: 3 dots (온보딩 슬라이드)
- CTA 버튼: Primary, lg
- 텍스트 링크: Ghost

**Interaction**:
- 좌우 스와이프: 온보딩 페이지 전환 (3장)
  - 1: "출처가 명확한 맛집 발견" + 일러스트
  - 2: "폴더로 체계적 관리" + 일러스트
  - 3: "교차 검증으로 신뢰" + 일러스트
- "시작하기" 탭: AuthModal(회원가입) 열림
- "로그인" 탭: AuthModal(로그인) 열림

**Empty/Error**: 해당 없음 (정적 화면)

```text
+---------------------------------------+
| [≡] 맛탐정    [위치: 강남] [🔍 검색]   |  ← Header
+---------------------------------------+
| [추천]  [소스]  [구독]                  |  ← 상단 탭
+---------------------------------------+
|                                       |
| 🔥 지금 뜨는 맛집 (섹션 제목)            |
|                                       |
| +---+  +---+                          |
| | PlaceCard | PlaceCard |             |
| +---+  +---+                          |
|                                       |
| 📍 강남 근처 추천 (섹션 제목)            |
|                                       |
| +---+  +---+                          |
| | PlaceCard | PlaceCard |             |
| +---+  +---+                          |
|                                       |
|        (무한 스크롤)                    |
+---------------------------------------+
| [홈]   [피드]  [맛탐정]  [프로필]        |
+---------------------------------------+
```

**Component Inventory**:
- Header: 로고, 위치 칩, 검색 아이콘
- 상단 탭 바: 추천/소스/구독
- 섹션 제목: Title 1

- 탭 전환: 추천 → 소스 탐색 (DiscoverGrid) → 구독 피드
- PlaceCard 탭: PlaceDetailModal 열림
- Pull-to-refresh: 추천 데이터 갱신
- 스크롤 다운: Bottom Nav 숨김 (y:60 slide out)
- 스크롤 업: Bottom Nav 복귀

**Empty State (구독 탭)**:

```text
+---------------------------------------+
|                                       |
|     [아이콘: Compass]                  |
|                                       |
|   아직 구독한 소스가 없어요              |
|   맛탐정을 탐색해 보세요                 |
|                                       |
|   +-----------------------------+     |
|   |       탐색하기 (Primary)      |     |
|   +-----------------------------+     |
|                                       |
+---------------------------------------+
```

**Loading**: PlaceCard 스켈레톤 (이미지 영역 + 텍스트 2줄)

> **Designer's Note**: 홈의 "추천" 탭은 신규 사용자의 첫 인상. 출처 배지가 카드에 항상 보여야 "이 앱은 출처가 다르다"는 인식을 심어줌. 네이버 지도처럼 별점을 보여주는 자리에 출처 배지를 배치하는 것이 핵심 전략.

맛탐정의 primary task = "출처 기반 장소 탐색". 소스 상세가 이를 대표함.

```text
+---------------------------------------+
| < 뒤로         공유  구독              |  ← DetailHeader

| [Avatar 64px]                         |
| 성시경의 먹을텐데  (Title 1)            |
| 유튜브 · 구독자 1.2K · 장소 87개       |
+---------------------------------------+
| [전체]  [방문한 곳만]                   |  ← VisitedFilterTab
+---------------------------------------+
| 정렬: 최신순 ▾         [지도 보기]      |
+---------------------------------------+
|                                       |
| +----------------------------------+  |
| | PlaceCard (compact)              |  |
| | 이미지 | 장소명, 지역, 카테고리     |  |
| |        | 출처 배지, ♡ ⊞           |  |
| +----------------------------------+  |
|                                       |
| +----------------------------------+  |
| | PlaceCard (compact)              |  |
| +----------------------------------+  |
|                                       |
|        (무한 스크롤)                    |
+---------------------------------------+
```

**Component Inventory**:
- DetailHeader: 뒤로, 공유, 구독 버튼
- 소스 프로필: Avatar(lg), Title, Meta(소스 타입, 구독자, 장소 수)
- VisitedFilterTab: 전체/방문한 곳만
- 정렬 드롭다운: 최신순/거리순
- PlaceCard (compact variant): 가로형 레이아웃
- 지도 보기 링크

**Interaction**:
- 구독 버튼 탭: 낙관적 업데이트, 토글
- 방문 필터 탭: 장소 목록 재조회
- 정렬 변경: 목록 재정렬
- PlaceCard 탭: PlaceDetailModal
- 지도 보기: 지도 페이지 이동

**Empty State**:

```text
+---------------------------------------+
|                                       |
|   아직 등록된 장소가 없어요              |
|                                       |
+---------------------------------------+
```

**Loading**: CompactPlaceCard 스켈레톤 × 5

### Screen 4: Detail View (PlaceDetailModal)

```text
+---------------------------------------+
| ─── (drag handle)                     |
+---------------------------------------+
| [이미지 갤러리 (16:9)]                  |
|  < > (이미지 넘기기)                    |
+---------------------------------------+
| 을지로 골뱅이집 (Title 1)               |
| 한식 · 을지로3가 · 영업 중              |
| 📍 서울 중구 을지로 ○○길               |
+---------------------------------------+
| ─── 출처 하이라이트 ───                 |
| 🔴 유튜브 2   🔵 커뮤니티 1            |
| 📁 폴더 3     👥 저장 12명             |
+---------------------------------------+
| ♡ 좋아요  ⊞ 저장  N 네이버  … 더보기   |  ← PlaceActionRow
+---------------------------------------+
| [메뉴] [리뷰] [출처]                    |  ← 탭
+---------------------------------------+
|                                       |
| (탭별 콘텐츠)                          |
|                                       |
| 메뉴 탭: 대표 메뉴·가격                 |
| 리뷰 탭: 댓글 목록 + 작성               |
| 출처 탭: 유튜브 링크, 폴더, 커뮤니티     |
|                                       |
+---------------------------------------+
```

**Component Inventory**:
- Drag handle (모달 상단)
- Image gallery (swipe, indicator dots)
- Title 영역: 장소명, 카테고리, 지역, 영업 상태
- Source Highlight: 출처 배지 그룹 (컬러 코딩)
- PlaceActionRow: 좋아요, 저장, 네이버, 더보기
- Content Tab bar: 메뉴/리뷰/출처
- 탭별 콘텐츠 영역

**Interaction**:
- Drag down: 모달 dismiss
- 이미지 좌우 스와이프: 갤러리 넘기기
- 좋아요 탭: 낙관적 업데이트, 하트 빨간색 채움 + haptic
- 저장 탭: SaveToCollectionSheet 열림
- 네이버 탭: 외부 브라우저로 네이버 지도 이동
- 더보기 탭: PlaceActionSheet 열림
- 출처 배지 탭: 유튜브 → 외부 링크, 폴더 → 폴더 상세 이동
- 탭 전환: 메뉴/리뷰/출처 콘텐츠 전환

**Error State**: 네트워크 오류 시 "장소 정보를 불러올 수 없어요" + 재시도

**Loading**: 이미지 스켈레톤 + 텍스트 3줄 스켈레톤 + 출처 배지 스켈레톤

> **Designer's Note**: 출처 하이라이트를 액션 바 위에 배치한 이유는 Interest Hierarchy 순서 때문. 음식 종류·지역(제목 영역) → 출처(하이라이트) → 액션(버튼). 사용자가 "관심 판단 → 행동"으로 자연스럽게 이어지도록 설계. 다른 앱에서 별점이 들어가는 위치에 출처 요약을 넣은 것이 맛탐정만의 패턴.

#### Step 2: Success Toast

```text
+---------------------------------------+
|                                       |

|  +--------------------------------+   |
```

**Component Inventory**:
- Sheet header: 제목, 닫기 버튼

- Success toast: 완료 메시지 + "보기" 액션

**Interaction**:

## 4. COMPONENT SPECIFICATIONS

### 4.1 Button Hierarchy

| 계층 | Variant | 용도 | 시각적 무게 |
|------|---------|------|-------------|
| Primary | bg-primary-600, text-white | CTA: 저장, 구독, 시작하기 | ████████ |
| Secondary | bg-surface-100, text-surface-700 | 보조 액션: 취소, 닫기 | ████░░░░ |
| Tertiary (Ghost) | transparent, text-surface-600 | 툴바, 시트 내 액션 | ██░░░░░░ |
| Destructive | bg-accent-rose, text-white | 삭제, 해제 | ████████ |
| Link | transparent, text-primary-600 | 텍스트 링크, "더보기" | █░░░░░░░ |

**규칙**: 화면당 Primary CTA는 1개만. "무엇을 해야 하는가"가 즉시 명확해야 함.

### 4.2 Form Patterns

#### Validation

| 시점 | 방식 |
|------|------|
| 입력 중 | 최소 길이 미만이면 비활성(disabled) 유지 |
| 포커스 아웃 | 유효성 검사 실행 |
| 제출 시 | 전체 검증, 첫 번째 오류 필드로 스크롤 |

#### Error Messaging

```text
[입력 필드]
  └── "닉네임은 2~20자여야 합니다" (Caption, error 색상)
```

- 에러 메시지는 필드 바로 아래
- border가 error 색상으로 변경
- 구체적 원인 명시 ("이미 사용 중인 닉네임입니다")

#### Success State

- 폼 제출 성공: toast + 이전 화면 복귀
- 인라인 성공: 체크 아이콘 + "저장됨" 텍스트 (1초 후 사라짐)

### 4.3 Card Layouts & Content Prioritization

#### PlaceCard Variants

| Variant | 레이아웃 | 용도 |
|---------|----------|------|
| Standard | 세로 (이미지 상단, 텍스트 하단) | 홈 추천, 그리드 |
| Compact | 가로 (이미지 좌, 텍스트 우) | 피드, 검색 결과, 소스 상세 |
| Minimal | 텍스트만 (이미지 없음) | 폴더 내 장소, 좁은 리스트 |

#### FolderCard Variants

| Variant | 레이아웃 | 용도 |
|---------|----------|------|
| Standard | 썸네일 그리드(4장), 제목, 장소 수, 구독자 | 소스 탐색 |
| Compact | 아이콘 + 제목 + 장소 수 | 폴더 선택 시트 |

| 조합 | Contrast Ratio | WCAG AA |
|------|---------------|---------|
| Text Primary (#171717) on Background (#FAFAFA) | 17.4:1 | ✅ |
| Text Secondary (#525252) on Background (#FAFAFA) | 7.4:1 | ✅ |
| Primary (#7C3AED) on White (#FFFFFF) | 5.3:1 | ✅ |
| Caption (#737373) on White (#FFFFFF) | 4.6:1 | ✅ |
| Error (#EF4444) on White (#FFFFFF) | 4.0:1 | ⚠️ (큰 텍스트만) |

### 5.4 Reduce Motion Alternatives

| 기본 | 모션 감소 시 |
|------|-------------|

| scale-in (0.15s) | 즉시 표시 |
| 모달 slide-up | 즉시 표시 |
| 스켈레톤 pulse | 정적 회색 블록 |

### 5.5 Focus Indicators

- `focus-visible:ring-2 ring-primary-500/40 ring-offset-2`
- 모든 인터랙티브 요소에 적용
- 탭 순서: Header → Content → Tab Bar (논리적 순서)

## 6. MICRO-INTERACTIONS

### 6.1 Transition Definitions

| 인터랙션 | Duration | Easing | 설명 |
|----------|----------|--------|------|
| 좋아요 토글 | 150ms | ease-out | 스케일 1→1.2→1 + 색상 변경 |
| 저장 토글 | 150ms | ease-out | 스케일 1→1.1→1 |
| 모달 열기 | 250ms | ease-out (cubic-bezier(0.25, 0.1, 0.25, 1)) | slide-up from bottom |
| 모달 닫기 | 200ms | ease-in | slide-down |
| 시트 열기 | 300ms | spring(damping: 0.8) | slide-up with bounce |
| 탭 전환 | 150ms | ease-out | opacity cross-fade |
| Toast 진입 | 200ms | ease-out | slide-up + fade-in |
| Toast 퇴장 | 150ms | ease-in | fade-out |
| Pull-to-refresh | — | spring | 회전 애니메이션 |
| 스크롤 시 네비 숨김 | 200ms | ease-out | translateY(60px) |

### 6.2 Haptic Feedback Mapping

| 인터랙션 | 햅틱 | 강도 |
|----------|------|------|
| 좋아요 탭 | Impact (light) | 약 |
| 저장 완료 | Notification (success) | 중 |
| 삭제 확인 | Notification (warning) | 중 |
| Long press 시작 | Impact (medium) | 중 |
| Pull-to-refresh 트리거 | Impact (light) | 약 |
| 에러 발생 | Notification (error) | 강 |

### 6.3 Sound Design

- 맛탐정은 소리를 사용하지 않음. 햅틱과 시각적 피드백에 의존.
- 이유: 공공장소(식당)에서 주로 사용. 소리는 사용자 경험을 방해할 수 있음.

## 7. RESPONSIVE BEHAVIOR

### 7.1 Breakpoint Adaptations

| Breakpoint | Layout | Navigation | Card |
|------------|--------|------------|------|
| Mobile (375~428px) | 1열, 풀와이드 카드 | Bottom Tab Bar | Standard/Compact |
| Tablet (768px) | 2열 그리드 | Bottom Tab Bar + 상단 탭 확장 | Standard 2열 |
| Desktop (1440px) | max-width 512px 컨테이너 | Sidebar + Content | 모바일과 동일 |

### 7.2 Orientation Change

| 방향 | 대응 |
|------|------|
| Portrait (기본) | 1열 레이아웃 |
| Landscape | 2열 그리드, 모달은 좌우 여백 확대, max-width 유지 |

### 7.3 Foldable Device

- Inner display: 태블릿 레이아웃 적용 (2열)
- Outer display: 모바일 레이아웃 유지
- Flex panel: 장소 리스트 (좌) + 장소 상세 (우) 분할 뷰 고려

> **Designer's Note**: 데스크톱에서 max-width 512px 컨테이너를 유지하는 것은 의도적 결정. 맛탐정은 모바일 퍼스트 앱이며, 데스크톱에서도 모바일과 동일한 경험을 제공하여 개발·유지보수 비용을 줄임. 태블릿은 2열 그리드로 화면을 활용하되, 상호작용 패턴은 동일하게 유지.
