# 소스 생태계와 교차 검증

## 1. 소스(Source)의 정의

**소스** = 맛집 정보의 출처. "이 음식점을 누가, 어디서, 어떻게 추천했는지"를 나타내는 단위임.

```text
소스 타입별 특성
-----------------
유튜브    : 영상으로 분위기·음식 생생 전달, 채널별 신뢰도
인스타    : 사진·위치 태그, 지역 인플루언서
커뮤니티  : 익명 검증, 실사용자 경험담
네이버    : 공식 정보, 리뷰(단 신뢰도 이슈)
맛탐정    : 개인 큐레이션, 테마별 정리
```

## 2. 소스의 공공재 성격

유튜브·인스타그램·커뮤니티 소스는 **플랫폼 관리자가 지속 추적**하여 DB화함.
한 번 수집된 데이터는 모든 사용자가 활용하는 **공공재**가 됨.

- 관리자: 채널/계정 모니터링, 새 콘텐츠에서 장소 추출
- 사용자: 별도 수고 없이 검색·구독만으로 접근

## 3. 교차 검증 메커니즘

하나의 음식점에 여러 소스가 연결될수록 신뢰도가 올라감.

```text
소스 1개  : "한 군데서만 봤는데..."
소스 2~3개: "몇 군데서 추천하네, 괜찮을 것 같아"
소스 4개+ : "여러 데서 검증됐다, 가볼 만해"
```

### 교차 검증의 효과

1. **객관성**: 한 소스의 주관을 넘어선 다중 검증
2. **정보 풍부함**: 유튜브(영상) + 커뮤니티(후기) + 인스타(사진) = 입체적 정보
3. **광고 회피**: 마케팅 리뷰와 달리, 다양한 독립적 소스의 합의

## 4. 맛탐정 소스의 특수성

맛탐정 폴더는 **사용자 생성 소스**임.
유튜브·커뮤니티와 달리, 누구나 큐레이터가 되어 자신만의 소스를 만들 수 있음.

- 구독자 수 → 영향력 지표
- 꾸준한 업데이트 → 신뢰도 축적
- 테마별 폴더 → "이 사람 취향이 나랑 비슷해" 매칭

## 5. 소스 확장 전략

| 단계 | 대상 | 방식 |
|------|------|------|
| 현재 | 유튜브, 인스타, 커뮤니티, 네이버, 맛탐정 | 수동·반자동 수집 |
| 확장 | 블로그, 쓰레드, 틱톡 | API·크롤링 확대 |
| 미래 | 사용자 직접 등록 | UGC 소스 품질 관리 |
