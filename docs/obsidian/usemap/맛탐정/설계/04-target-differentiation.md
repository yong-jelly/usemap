# 타겟 사용자 및 핵심 차별점

## 1. 타겟 사용자

1. **맛집 정보를 열심히 모으지만 관리가 안 되는 사람**
2. **광고성 리뷰가 아닌 진짜 추천을 찾는 사람**
3. **좋아하는 유튜버의 맛집 정보를 한 곳에서 보고 싶은 사람**
4. **친구/동료와 맛집 리스트를 공유하고 싶은 사람**
5. **여행 시 특정 지역 맛집을 효율적으로 탐색하고 싶은 사람**

## 2. 네이버 지도 대비 핵심 차별점

| 항목 | 네이버 지도 | 맛탐정 |
|------|-------------|--------|
| 저장 | 별표시만 가능 | 폴더 + 메모 + 출처 추적 |
| 공유 | 제한적 | 협업 편집, 초대 코드, 구독 |
| 추천 소스 | 광고 + 이벤트 리뷰 | 인플루언서, 커뮤니티, 맛탐정 |
| 신뢰도 | 마케팅 리뷰 | 교차 검증된 다중 소스 |
| 큐레이션 | 없음 | 맛탐정 테마별 폴더 |
| 영상 정보 | 없음 | 유튜브 링크로 생생한 정보 |
| 지역별 리스팅 | 어려움 | 지역/소스별 필터링 |
| 계정 | 네이버 종속 | 독립적 데이터 관리 |
| 사용자 역할 | 소비자 | 소비자 + 큐레이터(맛탐정) |
| 추천 근거 | 유저 리뷰·별점 | 출처 정보 (리뷰/별점 없음) |

### 핵심 콘셉트: 출처 기반 발견

> 타 서비스는 유저 리뷰·별점으로 음식점을 소개함. 맛탐정은 **리뷰/별점 없음**. 출처 정보(유튜브, 커뮤니티, 맛탐정 폴더 등)에 의해 관심·흥미를 가질 음식점을 발견·탐색을 위해 나열함.

## 3. 디자인 원칙

1. **Actionable**: 모든 정보는 "나중에 방문할 때" 도움이 되어야 함
2. **Traceable**: 모든 정보는 "어디서 왔는지" 추적 가능해야 함
3. **Personal**: 남의 취향보다 "나의 메모와 나의 맥락"이 우선
4. **Reliable**: 다양한 소스의 교차 검증으로 "나쁘지 않다는 확신" 제공
