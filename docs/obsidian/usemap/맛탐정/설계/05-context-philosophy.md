# 맥락 보존 철학

## 1. 왜 맥락이 중요한가

맛집 저장의 본질은 "나중에 그 근처에 갔을 때 기억해내는 것"임.
장소 이름·주소만 남기면, 저장 당시의 "왜 저장했는지"가 사라짐.

```text
저장 시점                          활용 시점 (몇 달 후)
---------                          ------------------
"유튜브에서 본 그 국밥집"     -->   "여기 왜 저장했지?"
"친구가 회식 추천해준 곳"    -->   "누가 추천했더라?"
"분위기 좋은 데이트 코스"    -->   "이거 데이트용이었나?"
```

**맥락이 있으면**: "아, 유튜브에서 본 그 국밥집이구나. 영상 다시 보면서 메뉴 확인해볼게."
**맥락이 없으면**: "뭔지 모르겠네. 그냥 근처 다른 데 가자."

## 2. 맥락의 구성 요소

| 요소 | 설명 | 맛탐정에서의 구현 |
|------|------|------------------|
| 출처(Source) | 어디서 발견했는지 | 유튜브/인스타/커뮤니티/맛탐정 폴더 자동 기록 |
| 저장 이유 | 왜 저장했는지 | 메모·태그·폴더명으로 표현 |
| 시점 | 언제 저장했는지 | 저장일 자동 기록 |
| 맥락 링크 | 원본 콘텐츠 | 유튜브 영상, 인스타 게시물 링크 보존 |

## 3. 미식 아카이브 메타포

맛탐정은 SNS가 아님. "나만의 미식 지식 베이스"에 가깝게 설계함.

- **SNS**: 타인의 반응, 팔로우, 트렌드가 중심
- **아카이브**: 나의 경험, 나의 맥락, 나의 기준이 중심

즉, "다른 사람에게 보여주기"보다 "나중에 나에게 도움이 되기"를 우선함.

## 4. Slack '나중에' 감성

Slack의 "나중에" 기능처럼, "지금은 처리 못하지만 나중에 꼭 활용할 정보"를 안전하게 보관하는 도구적 가치를 추구함.

- **저장 시**: 부담 없이 빠르게 저장 (출처 자동 기록)
- **활용 시**: 그 근처에 갔을 때, 저장 당시 맥락을 복구하여 의사결정 지원
