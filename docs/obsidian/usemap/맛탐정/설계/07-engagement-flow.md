# 음식점 상세에서의 관심 계기와 액션 흐름

## 1. 핵심 콘셉트

> **맛탐정은 리뷰/별점 없음. 출처 정보에 의해 유저가 관심·흥미를 가질 음식점을 발견·탐색을 위해 나열해줌.**

타 서비스(네이버, 다이닝코드 등)는 유저 리뷰·별점을 바탕으로 음식점을 소개함.
맛탐정은 **출처 정보**를 바탕으로 "이 음식점에 관심 가질 만한가"를 판단하게 함.

## 2. 관심 계기 순서 (Interest Hierarchy)

음식점 상세에서 유저가 관심을 가지게 되는 계기는 아래 순서로 작동함.

```text
1. 음식 컬러(종류)     : 한식/일식/중식/양식 등 - "내가 먹고 싶은 종류인가"
2. 지역                : "지금 가기 좋은 위치인가"
3. 출처(Source)        : "어디서 추천됐는가" - 신뢰의 근거
   ├── 출처 개수       : 몇 개의 소스에서 언급됐는가
   ├── 콜렉션 저장 수  : 몇 개의 콜렉션(폴더)에 저장됐는가
   ├── 콜렉션 이름     : 어떤 테마의 콜렉션에 포함됐는가
   └── 저장 인원 수   : 몇 명이 저장했는가
4. 댓글 여부           : 긍정/부정, 개수 - 맛탐정 내 소통
5. 네이버 별점/리뷰    : 네이버 지도 버튼으로 외부 확인 (맛탐정 내부 아님)
```

### 우선순위 해석

- **음식 종류·지역**이 먼저: "이걸 먹고 싶다", "여기가 편하다"가 선행 조건
- **출처**가 핵심: 맛탐정의 차별점. 유튜브/커뮤니티/맛탐정 폴더 등 "왜 믿을 수 있는가"
- **콜렉션 저장**이 출처보다 구체적: "어떤 맥락에서 저장됐는가" (데이트용, 회식용 등)
- **댓글**은 보조: 맛탐정 내 사용자 반응
- **네이버**는 외부 참고: 맛탐정이 직접 제공하지 않고, 버튼으로 연결만 함

## 3. 액션 순서 (Action Hierarchy)

관심이 충족되면 유저가 취하는 액션 순서.

```text
1. 좋아요 (Like)     : 콜렉션 고민 없이 가볍게 표시. "관심 있음"만 남김
2. 콜렉션 저장       : "어느 폴더에 넣을지" 결정 후 저장
```

### 좋아요 vs 콜렉션 저장

| 구분 | 좋아요 | 콜렉션 저장 |
|------|--------|-------------|
| 부담 | 낮음 (클릭 한 번) | 높음 (폴더 선택 필요) |
| 용도 | 관심 표시, 나중에 정리 | 맥락별 분류, 메모 추가 |
| 흐름 | 관심 → 좋아요 → (나중에) 콜렉션 저장 |

"콜렉션에 넣을까 말까" 고민할 필요 없이 먼저 좋아요로 관심을 표시하고, 나중에 콜렉션으로 정리하는 흐름을 지원함.

## 4. 리뷰·별점의 포지셔닝

- **기능 없음 또는 소극적 제공**
- 맛탐정의 핵심은 "출처 기반 발견"이지 "리뷰 기반 추천"이 아님
- 네이버 지도 버튼으로 별점·리뷰는 외부에서 확인하도록 유도
