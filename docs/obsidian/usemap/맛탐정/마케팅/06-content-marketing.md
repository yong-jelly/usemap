# 6. CONTENT MARKETING (5 assets)

## 6.1 Blog Post Outlines (3개)

### Blog 1: "왜 우리는 별점을 없앴나 — 출처 기반 발견의 철학"

**목적**: 브랜드 스토리, SEO (맛집 앱, 맛집 추천, 별점 없는 맛집)
**대상**: 맛집에 관심 있는 일반 사용자 + 프로덕트에 관심 있는 PM/디자이너

```text
1. 도입: 네이버 별점 4.5의 실체 (경험 공유)
   · 별점 높은 음식점 → 실망 경험
   · 체험단·마케팅 리뷰의 비중
   · "진짜 추천이 어디 있는가?"

2. 문제 정의: 리뷰·별점의 구조적 한계
   · 별점은 평균이지, 나에게 맞는 추천이 아님
   · 광고 인센티브가 리뷰 품질을 떨어뜨림
   · 리뷰를 읽는 시간 대비 정보 가치

3. 대안: 출처 기반 발견
   · "누가 추천했는가"가 "몇 점인가"보다 중요
   · 유튜브 영상 하나 > 텍스트 리뷰 수십 개
   · 교차 검증: 여러 독립 소스의 합의

4. 맛탐정에서의 구현
   · 출처 하이라이트 영역
   · 소스 배지 컬러 코딩
   · PlaceCard에서 별점 자리에 출처 배지

5. 결론: "출처가 다르면, 발견이 다르다"
```

- **CTA**: 맛탐정에서 출처 기반 맛집 발견하기
- **SEO 키워드**: 맛집 추천, 별점 없는 맛집 앱, 맛집 출처, 교차 검증
- **예상 분량**: 2,000~2,500자

### Blog 2: "맛집 저장의 비극 — 왜 300개 별표 중 하나도 못 꺼내 쓸까"

**목적**: Pain point 공감, SEO (맛집 저장, 맛집 관리, 맛집 폴더)
**대상**: 맛집 저장을 많이 하는 사용자

```text
1. 도입: "여기 왜 저장했지?" 경험
   · 유튜브에서 본 국밥집 → 3개월 후 기억 없음
   · 네이버 별표 300개 → 여행 시 활용 불가
   · 결국 "강남 맛집" 재검색

2. 원인 분석: 맥락 소실 (Context Loss)
   · 저장 도구는 '장소 이름'만 남김
   · 출처(누가 추천?), 이유(왜 저장?), 맥락(데이트용?) 소실
   · Slack '나중에' 비유: 맥락 없는 '나중에'는 '영원히'가 됨

3. 해결: 맥락과 함께 저장하기
   · 폴더: "데이트 코스", "부산 여행" 등 테마 분류
   · 메모: "유튜브에서 본 파스타 맛집, 런치 세트 추천"
   · 출처 자동 기록: 어디서 발견했는지 추적

4. 실전 팁: 맛집 폴더 정리법
   · 상황별 폴더 만들기
   · 메모 습관: 한 줄이면 충분
   · 좋아요 → 폴더 정리 2단계 활용

5. 결론: "나만의 미식 아카이브"
```

- **CTA**: 맛탐정에서 첫 폴더 만들기
- **SEO 키워드**: 맛집 저장, 맛집 관리, 맛집 폴더, 맛집 정리
- **예상 분량**: 2,000자

### Blog 3: "유튜버 맛집, 영상 속에서 어떻게 꺼내나 — 유튜브 맛집 데이터베이스"

**목적**: 유튜브 맛집 검색 니즈 타겟, SEO (유튜브 맛집, 유튜버 추천 맛집)
**대상**: 유튜브 먹방 시청자

```text
1. 도입: "성시경이 갔던 그 국밥집 어디였지?"
   · 인기 먹방 채널 영상 수백~수천 개
   · 과거 영상 하나씩 찾아보기 비현실적
   · 유튜브 맛집 정보의 휘발성

2. 문제: 유튜브 맛집 정보의 접근성 한계
   · 영상 단위 ≠ 장소 단위
   · 검색 어려움 (영상 제목에 장소명 없는 경우)
   · 메모 도구 없음

3. 솔루션: 채널별 맛집 데이터베이스
   · 맛탐정이 유튜브 채널별로 장소 추출·정리
   · 채널 이름으로 검색 → 장소 리스트
   · 원본 영상 링크 보존

4. 활용법
   · "성시경" 검색 → 87개 장소 → 지역 필터 → 저장
   · 소스 구독 → 새 맛집 추가 시 피드 알림
   · 교차 검증: 유튜브 + 커뮤니티가 겹치는 곳 찾기

5. 결론: 유튜브 맛집을 놓치지 마세요
```

- **CTA**: 맛탐정에서 유튜버 맛집 검색하기
- **SEO 키워드**: 유튜브 맛집, 유튜버 추천 맛집, 먹방 맛집, 성시경 맛집
- **예상 분량**: 1,800자

## 6.2 Whitepaper 구조

### "출처 기반 발견: 맛집 추천의 새로운 패러다임"

**목적**: 업계 리더십, 투자자/파트너 신뢰 구축
**대상**: 투자자, 업계 관계자, PM/디자이너
**분량**: 8~12페이지

```text
1. Executive Summary (1p)
   · 출처 기반 발견의 정의와 가치

2. 시장 분석 (2p)
   · 한국 맛집 정보 시장 현황
   · 리뷰·별점 기반 서비스의 한계
   · 유튜브 먹방 콘텐츠 시장 성장
   · 사용자 행동 변화 (리뷰 → 영상)

3. 문제 정의 (2p)
   · 리뷰 조작·광고 리뷰 실태
   · 맥락 소실 문제
   · 유튜브 맛집 정보 접근성

4. 솔루션: 출처 기반 발견 모델 (3p)
   · 소스(Source) 데이터 모델
   · 교차 검증 메커니즘
   · 맛탐정 생태계 (큐레이터 경제)
   · 맥락 보존 설계 철학

5. 데이터 & 인사이트 (2p)
   · 소스 유형별 교차 검증 현황
   · 사용자 저장·활용 패턴
   · 교차 검증 수와 재방문율 상관관계

6. 로드맵 (1p)
   · 소스 확장 전략
   · 비즈니스 모델 진화
   · 기술 로드맵

7. 결론 & CTA (1p)
```

## 6.3 Webinar Script Outline

### "맛집 추천, 별점을 넘어서 — 출처 기반 발견의 현재와 미래" (45분)

**목적**: 브랜드 인지도, 얼리어답터 확보
**대상**: 맛집에 관심 있는 일반 사용자 + 푸드테크 관계자

```text
[0:00~5:00] 오프닝
· 자기소개
· "최근에 맛집 검색하면서 실망한 경험 있으신 분?"
· 오늘 다룰 내용 소개

[5:00~15:00] Part 1: 맛집 추천의 현재
· 네이버 별점의 구조적 문제 (데이터 기반)
· 유튜브 먹방의 영향력과 한계
· 사용자 행동 변화: 리뷰 읽기 → 영상 확인

[15:00~25:00] Part 2: 출처 기반 발견이란
· "누가 추천했는가" > "몇 점인가"
· 교차 검증의 원리와 효과
· 맛탐정 라이브 데모
  - 장소 검색 → 출처 확인
  - 소스 탐색 → 유튜버별 맛집
  - 폴더 저장 → 메모 작성

[25:00~35:00] Part 3: 맛탐정 활용법
· 폴더 정리 실전 팁
· 협업 폴더 (회식, 여행, 모임)
· 맛탐정 되기: 공개 폴더로 큐레이션 시작

[35:00~45:00] Q&A & 클로징
· 참석자 질문 응답
· 맛탐정 가입 방법 안내
· 특별 혜택 (얼리어답터 배지 등)
```
