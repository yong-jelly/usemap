# 3. LANDING PAGE COPY (5 assets)

## 3.1 Hero Section

### Variation A: 출처 강조 (추천)

```text
+-----------------------------------------------+
|                                               |
|  출처가 다르면,                                |
|  발견이 다르다.                                |
|                                               |
|  별점 대신 유튜브·커뮤니티·인스타 출처로         |
|  교차 검증된 맛집을 발견하세요.                  |
|                                               |
|  [맛탐정 시작하기]   [어떻게 다른지 보기 ↓]     |
|                                               |
+-----------------------------------------------+
```

- **Headline**: Display, Bold, Primary 또는 Foreground
- **Subheadline**: Body, muted
- **CTA**: Primary(시작하기) + Ghost(스크롤 유도)
- **Visual**: 장소 카드 UI 목업 + 출처 배지 강조 (floating)

### Variation B: Pain point 시작

```text
+-----------------------------------------------+
|                                               |
|  맛집 300개 저장해놓고,                         |
|  결국 재검색하셨죠?                             |
|                                               |
|  출처와 메모를 함께 저장하면                     |
|  3개월 뒤에도 "왜 저장했는지" 기억납니다.         |
|                                               |
|  [지금 정리 시작하기]                           |
|                                               |
+-----------------------------------------------+
```

**A/B 테스트**: Variation A(출처 차별점) vs B(Pain point). 전환율 기준.

## 3.2 Feature Sections (3 variations)

### Feature 1: 출처 기반 발견

```text
━━━ 별점 없이, 출처로 판단하세요 ━━━

[장소 상세 UI: 출처 하이라이트 영역]

하나의 음식점에 유튜브, 커뮤니티, 인스타 출처가 연결됩니다.
여러 소스가 추천한 곳일수록 신뢰도가 올라갑니다.

• 유튜브 — 영상으로 분위기와 음식을 생생하게 확인
• 커뮤니티 — 익명 실사용자의 솔직한 경험담
• 인스타그램 — 사진과 위치 태그로 빠른 탐색
• 맛탐정 폴더 — 테마별 큐레이션, 구독 가능

출처가 3개 이상이면? → 가볼 만합니다.
```

- **Visual**: 출처 배지 컬러 코딩, 숫자가 증가하는 애니메이션
- **Layout**: 좌측 텍스트 + 우측 UI 스크린샷 (모바일에서는 상하)

### Feature 2: 폴더로 체계적 관리

```text
━━━ 별표 말고, 폴더로 정리하세요 ━━━

[폴더 목록 UI]

테마별 폴더에 맛집을 저장하세요.
메모와 출처가 함께 남아, 나중에도 맥락이 살아있습니다.

• "데이트 코스" — 분위기 좋은 레스토랑 모음
• "회식 맛집" — 팀원과 공유하는 협업 폴더
• "부산 여행" — 여행지별 맛집 정리

폴더를 공개하면 누구나 구독할 수 있어요.
나만의 큐레이션을 시작해 보세요.
```

- **Visual**: 폴더 카드 3개 + 메모가 보이는 상세 화면
- **Layout**: 중앙 정렬, 폴더 카드가 살짝 겹치는 구성

### Feature 3: 교차 검증 시스템

```text
━━━ 여러 소스가 추천하면 믿을 수 있습니다 ━━━

[교차 검증 다이어그램]

      유튜브 ──┐
               ├──→ 을지로 골뱅이집 ──→ 신뢰도 UP
   커뮤니티 ──┤
               ├──→ 교차 검증 완료
  인스타그램 ──┘

광고 리뷰와 다른, 독립적인 소스들의 합의.
텍스트 리뷰로는 알 수 없는 정보를
유튜브 영상으로 직접 확인할 수 있습니다.
```

- **Visual**: 소스들이 하나의 장소로 수렴하는 다이어그램
- **Layout**: 전체 너비, 다이어그램 중앙 배치

## 3.4 FAQ Section (8개)

| # | 질문 | 답변 |
|---|------|------|
| Q1 | 맛탐정은 무엇인가요? | 유튜브, 커뮤니티, 인스타그램 등 다양한 출처의 맛집 정보를 한곳에 모아 교차 검증하는 플랫폼입니다. 별점이나 리뷰 대신, 출처 정보로 맛집을 발견하고 폴더에 체계적으로 저장할 수 있어요. |
| Q2 | 왜 별점/리뷰가 없나요? | 맛탐정은 유저 리뷰·별점 기반이 아닌, 출처 기반 발견을 추구합니다. 유튜브 영상, 커뮤니티 후기 등 독립적인 소스의 교차 검증으로 광고 리뷰와 다른 객관성을 제공해요. 네이버 지도 버튼으로 별점/리뷰를 외부에서 확인할 수도 있습니다. |
| Q3 | 교차 검증이란 무엇인가요? | 하나의 음식점에 여러 소스(유튜브, 커뮤니티, 인스타 등)가 연결되는 것입니다. 소스가 많을수록 "여러 곳에서 검증됐다"는 의미이므로 신뢰도가 올라갑니다. |
| Q4 | 폴더는 어떻게 사용하나요? | 테마별로 폴더를 만들어 맛집을 저장할 수 있어요. "데이트 코스", "회식 맛집" 등 자유롭게 분류하고, 메모를 남겨 저장 이유를 기록해 보세요. 공개 폴더로 설정하면 다른 사람도 구독할 수 있습니다. |
| Q5 | "맛탐정"은 무슨 뜻인가요? | 공개 폴더를 운영하는 큐레이터를 뜻합니다. 누구나 자신만의 맛집 큐레이션을 시작하면 맛탐정이 될 수 있어요. 구독자가 늘면 미식 영향력이 쌓입니다. |
| Q6 | 무료인가요? | 네, 맛탐정의 모든 기능은 무료입니다. |
| Q7 | 어떤 소스가 있나요? | 현재 유튜브 채널, 인스타그램 계정, 커뮤니티(클리앙, 다모앙, 보배드림), 네이버 플레이스 폴더, 그리고 사용자가 만든 맛탐정 폴더가 있습니다. 지속적으로 소스를 확대하고 있어요. |
| Q8 | 친구와 함께 사용할 수 있나요? | 네. 폴더를 만들고 초대 코드를 공유하면 친구/동료와 함께 맛집을 추가하고 관리할 수 있습니다. |

## 3.5 Pricing Page

맛탐정은 현재 **전체 무료**이므로, Pricing 대신 "가치 비교" 섹션으로 대체.

```text
━━━ 맛탐정 vs 기존 방식 ━━━

| 기능            | 네이버 지도   | 맛탐정       |
|----------------|-------------|-------------|
| 저장            | 별표만        | 폴더+메모+출처 |
| 공유            | 제한적        | 협업 폴더, 구독 |
| 추천 근거        | 광고 리뷰     | 교차 검증 출처  |
| 유튜브 연결      | 없음         | 출처 영상 확인  |
| 지역별 탐색      | 어려움        | 필터링       |
| 비용            | 무료         | 무료         |

━━━━━━━━━━━━━━━━━━━━━━━━━━

    같은 무료인데, 출처가 다릅니다.

    [지금 시작하기] (Primary CTA)
```

- **Visual**: 비교 표, 맛탐정 열에 체크마크 강조
- **A/B**: 비교 표 vs 가치 카드 3장 레이아웃
