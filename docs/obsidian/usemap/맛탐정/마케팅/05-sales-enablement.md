# 5. SALES ENABLEMENT (7 assets)

맛탐정은 B2C 서비스이지만, 파트너십(유튜브 채널, 커뮤니티, 기업 제휴)과 투자 유치 시 활용 가능한 자료.

## 5.3 Case Study Template

### 구조

```text
━━━ Case Study: [소스명/파트너명] ━━━

■ 배경
· [소스/파트너]의 기존 상황
· 맛탐정과 연결하게 된 계기

■ 과제
· [해결하려던 문제]

■ 솔루션
· 맛탐정에서의 구체적 적용
· 소스 연결, 교차 검증 효과

■ 결과
· 구독자 수 변화
· 장소 저장 수 변화
· 사용자 피드백

■ 핵심 인용
· "[소스명] 맛집 정보가 맛탐정에 정리되니,
   팬들이 훨씬 쉽게 접근할 수 있게 됐어요."
```

### 예시 시나리오

- **유튜브 채널 케이스**: "성시경의 먹을텐데" 채널의 87개 장소가 맛탐정에 DB화 → 팬들이 채널별 검색으로 맛집 탐색 → 유튜브 영상 유입 증가
- **커뮤니티 케이스**: 클리앙 맛집게시판의 추천글이 자동 수집 → 커뮤니티 사용자가 맛탐정에서 교차 검증 확인 → 재방문율 증가

## 5.4 Battlecard (경쟁사 비교)

```text
━━━ 맛탐정 vs 경쟁사 Battlecard ━━━

| 기능 | 맛탐정 | 네이버 지도 | 다이닝코드 | 망고플레이트 | 캐치테이블 |
|------|--------|------------|-----------|------------|-----------|
| 추천 근거 | 출처 교차 검증 | 유저 별점·리뷰 | 알고리즘 | 유저 리뷰 | 예약 기반 |
| 별점 | 없음 | 있음 | 있음 | 있음 | 없음(예약) |
| 저장 방식 | 폴더+메모+출처 | 별표 | 위시리스트 | 가고싶다 | 예약 내역 |
| 유튜브 연결 | 출처 영상 링크 | 없음 | 없음 | 없음 | 없음 |
| 협업 | 공유 폴더, 초대 | 제한적 | 없음 | 없음 | 없음 |
| 구독 | 소스/폴더 구독 | 없음 | 없음 | 팔로우 | 없음 |
| 큐레이터 | 누구나 맛탐정 | 없음 | 없음 | 일부 | 없음 |
| 교차 검증 | 다중 소스 연결 | 없음 | 단일 알고리즘 | 없음 | 없음 |
| 광고 리뷰 | 없음 | 많음 | 일부 | 적음 | 없음 |
| 비용 | 무료 | 무료 | 무료 | 무료 | 예약 수수료 |
```

### 포지셔닝 차별점 요약

- **vs 네이버 지도**: "별점의 신뢰도 vs 출처의 교차 검증"
- **vs 다이닝코드**: "알고리즘이 아닌, 사용자가 직접 출처를 확인"
- **vs 망고플레이트**: "리뷰가 아닌, 유튜브·커뮤니티 소스 통합"
- **vs 캐치테이블**: "예약이 아닌, 발견과 저장에 집중"

### 반론 대응

| 경쟁사 주장 | 맛탐정 대응 |
|------------|-----------|
| "별점이 있어야 판단 가능" | 별점은 광고·이벤트 리뷰로 왜곡됨. 출처 개수가 더 객관적 |
| "리뷰가 풍부해서 편함" | 리뷰 읽는 시간 vs 출처 확인 시간. 유튜브 영상 하나가 리뷰 수십 개보다 정보량 높음 |
| "이미 네이버에 다 있음" | 네이버는 저장 맥락 소실. 폴더+메모+출처가 없음 |

## 5.5 Product Demo Script

```text
━━━ 맛탐정 제품 데모 (5분) ━━━

[0:00~0:30] 오프닝
"맛집 앱에서 별점을 없애면 어떤 일이 일어날까요?"

[0:30~1:30] 문제 제기
- 네이버 별점 4.5 → 체험단 리뷰 비율 보여주기
- 유튜브 맛집 영상 → "이 맛집 어디였지?" 시나리오
- 맛집 300개 저장 → "왜 저장했지?" 시나리오

[1:30~2:30] 솔루션: 출처 기반 발견
- 앱 열기 → 홈 화면 (추천 탭)
- PlaceCard 클릭 → 장소 상세
- 출처 하이라이트: "유튜브 2, 커뮤니티 3, 폴더 4"
- "별점 대신 이 출처를 봅니다"
- 유튜브 출처 클릭 → 영상 바로 확인

[2:30~3:30] 솔루션: 폴더로 관리
- 저장 버튼 → 폴더 선택 시트
- "데이트 코스" 폴더에 저장
- 메모 작성: "유튜브에서 본 파스타 맛집"
- 폴더 내 장소 리스트 보여주기

[3:30~4:15] 솔루션: 소스 탐색
- 맛탐정 탭 → 소스 목록
- "성시경의 먹을텐데" 클릭 → 87개 장소
- 구독 버튼 → "새 맛집이 추가되면 피드로 알림"

[4:15~5:00] 클로징
"출처가 다르면, 발견이 다릅니다.
맛탐정에서 별점 없이도 신뢰할 수 있는 맛집을 발견해 보세요."
```

## 5.6 Objection Handling Guide (10개)

| # | 반론 | 대응 |
|---|------|------|
| 1 | "별점 없으면 어떻게 판단해?" | 출처 개수 = 교차 검증 수준. 유튜브+커뮤니티+인스타가 동시에 추천했다면, 별점 4.5보다 신뢰할 수 있습니다. 유튜브 영상으로 직접 확인도 가능합니다. |
| 2 | "네이버 지도면 충분한데?" | 네이버는 저장은 되지만 '왜 저장했는지' 맥락이 사라집니다. 수백 개 별표 중에서 원하는 걸 찾기도 어렵고요. 맛탐정은 폴더+메모+출처로 맥락을 보존합니다. |
| 3 | "맛집 앱 너무 많지 않아?" | 기존 앱은 모두 리뷰·별점 기반입니다. 출처 기반 발견과 교차 검증은 맛탐정만의 접근입니다. |
| 4 | "소스가 충분히 많아?" | 유튜브 N개 채널, 인스타 N개, 커뮤니티 3개(클리앙, 다모앙, 보배드림), 네이버 폴더, 그리고 사용자 맛탐정 폴더까지. 지속 확대 중입니다. |
| 5 | "어떻게 수익을 내?" | 현재는 사용자 확보 단계. 프리미엄 폴더(고급 통계, 비공개 협업), 소스 파트너십(유튜브 채널 프로모션), 데이터 인사이트 제공 등을 검토 중입니다. |
| 6 | "유튜버가 직접 참여하나?" | 유튜버 참여 없이도 공개 영상의 맛집 정보를 DB화합니다. 유튜버 입장에서는 자기 맛집 목록이 정리되어 팬 접근성이 높아지는 장점이 있습니다. |
| 7 | "커뮤니티 글은 신뢰도가 낮지 않나?" | 단일 소스로는 낮을 수 있지만, 교차 검증이 핵심입니다. 커뮤니티+유튜브+인스타가 모두 추천한 곳이면 신뢰도가 올라갑니다. |
| 8 | "폴더 관리가 번거롭지 않나?" | 좋아요(가벼운 관심) → 나중에 폴더 정리로 2단계 플로우를 지원합니다. 폴더 없이도 좋아요만으로 관심 표시가 가능합니다. |
| 9 | "광고가 들어오면 같은 문제 아닌가?" | 맛탐정의 구조상 광고 리뷰가 들어갈 곳이 없습니다. 별점·리뷰가 없고, 출처만 표시하므로 출처 자체의 독립성이 보장됩니다. |
| 10 | "왜 지금 이 서비스가 필요해?" | 유튜브 먹방 콘텐츠 폭발적 증가 + 네이버 리뷰 신뢰도 하락이 동시에 일어나고 있습니다. "유튜브에서 본 맛집을 쉽게 찾고 저장하는 도구"에 대한 수요가 커지고 있습니다. |

## 5.7 Proposal Template

### 파트너십 제안서 구조

```text
━━━ 맛탐정 파트너십 제안서 ━━━

1. Executive Summary
   · 맛탐정 한 줄 소개
   · 파트너십 목적 및 기대 효과

2. 맛탐정 소개
   · 서비스 개요 (출처 기반 발견)
   · 핵심 수치 (사용자, 소스, 장소)

3. 파트너 현황 분석
   · [파트너] 채널/콘텐츠 현황
   · 맛탐정에서의 노출 현황

4. 제안
   · 파트너 전용 소스 페이지
   · 구독자 → 팬 전환 기회
   · 교차 검증 데이터 활용

5. 기대 효과
   · 파트너: 콘텐츠 접근성 향상, 팬 도달 확대
   · 맛탐정: 콘텐츠 품질 향상, 소스 확대

6. 다음 단계
   · 미팅 일정
   · 연락처
```
